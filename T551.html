Description

The custom status setting (from the user settings store) is the source of truth, and that's what's sent over the gateway socket. Other clients should still see the status, and it's also what powers the custom status setting UI, so that should be correct too.

However, the local presence store often doesn't correctly update, so other UI where you can view your own status (e.g., user list and profile sheet) can be incorrect.

This is a local, client-side only bug. If the status is not showing for other clients when it should be, then a new bug should be filed.

See dupes for other repro steps.
